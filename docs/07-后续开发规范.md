# 07-后续开发规范

## 代码规范

- TypeScript 严格模式。
- 页面放在 `src/app`。
- 可复用 UI 放在 `src/components`。
- 业务功能按领域放在 `src/features`。
- 工具函数放在 `src/lib`。

## 提交规范

使用 Conventional Commit：

```text
feat: add blog index
fix: correct nginx cache header
docs: update deployment guide
```

## AI 协作原则

- 每次变更保持小步提交。
- 优先修改高内聚模块，避免跨层大文件。
- 新增功能时同步更新 README 或 docs。
